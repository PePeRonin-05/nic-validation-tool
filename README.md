# NIC Validation Tool

Инструмент для нагрузочного тестирования 100G сетевых карт в режиме loopback.  
Два порта одной карты соединяются кабелем и тестируются на максимальную пропускную способность.

---

## Что тестирует

| Тест | Инструмент | Что измеряет |
|---|---|---|
| `iperf` | iperf3 | Двунаправленная пропускная способность |
| `rdma` | ib_read_bw + ib_write_bw | RDMA Read и Write одновременно |
| `all` | оба | iperf сначала, затем RDMA |

---

## Запуск

```bash
./run.sh <тест> <хост> <порт1> <порт2> [<порт3> <порт4> ...] <длительность-сек>
```

Длительность всегда **последний** аргумент. Пар портов можно указать сколько угодно.

**Примеры**

```bash
# Одна пара портов, 12 часов
./run.sh iperf 10.0.0.1 enp241s0f0np0 enp241s0f1np1 43200
./run.sh rdma  10.0.0.1 enp241s0f0np0 enp241s0f1np1 43200

# Две пары одновременно (две карты)
./run.sh rdma  10.0.0.1 ens121f0np0 ens121f1np1 ens120f0np0 ens120f1np1 43200

# Оба теста подряд, суммарно 24 часа (по 12 на каждый)
./run.sh all   10.0.0.1 enp241s0f0np0 enp241s0f1np1 86400
```

## Логи

После каждого запуска сохраняются локально в `local_logs/`:

```
local_logs/
├── iperf_43200s/iperf_loopback/   — логи iperf3
├── rdma_43200s/rdma_loopback/     — логи RDMA
└── all_86400s/all/                — логи обоих тестов
```

Каждый архив содержит:
- `iperf/` — вывод iperf3 клиента и сервера
- `rdma/` — вывод ib_read_bw / ib_write_bw
- `iface_status/` — статистика интерфейсов до и после
- `pci_logs/` — dmesg и journalctl по каждому интерфейсу

---

## Настройки

Файл `inventory/group_vars/all.yml`:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `ansible_ssh_pass` | `12345678` | Пароль root на целевом хосте |
| `test_mtu` | `9000` | MTU на тестовых интерфейсах |
| `iperf_parallel` | `4` | Параллельных потоков iperf3 |
| `rdma_qp` | `8` | Queue pairs для RDMA |
| `rdma_msg_size` | `16384` | Размер сообщения RDMA (байт) |

## Структура проекта

```
.
├── run.sh                      ← точка входа
├── site_loopback.yml           ← плейбук iperf3
├── site_rdma_loopback.yml      ← плейбук RDMA
├── site_all.yml                ← плейбук all (один setup на оба теста)
├── inventory/
│   ├── hosts.yml               ← пример хостов
│   └── group_vars/
│       └── all.yml             ← все настройки и пороги
└── roles/
    ├── common/                 ← назначение IP, MTU, ARP
    ├── iperf_loopback/         ← тест iperf3
    ├── rdma_loopback/          ← тест RDMA
    ├── parse_results/          ← парсинг логов и вывод результата
    ├── collect_logs/           ← архивирование и скачивание логов
    ├── diagnostics/            ← статистика интерфейсов
    └── cleanup/                ← завершение процессов, очистка IP и ARP
```
